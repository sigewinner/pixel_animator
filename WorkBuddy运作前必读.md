# WorkBuddy 运作前必读

在开始对本项目做任何修改之前，请先阅读本文件并严格遵守以下规则。

## 规则 1：改动文件后必须校对软件说明书

每次更改项目中的任意文件时，都要检查 **`PixelAnimator_软件架构与功能实现说明书.md`**（位于本工作文件夹 `D:\AAA_develop\github\pixel_animator-1.8-UI-\PixelAnimator_软件架构与功能实现说明书.md`，同目录）中记录的「功能 / 实现位置（文件+函数+行号）/ 原理」是否仍与文件实际内容对应。

- 若**对应**：无需改动说明书。
- 若**不对应**（例如：新增或删除功能、移动 / 重命名文件、修改函数逻辑、实现位置或行号发生变化等）：**必须同步更新该说明书**，使文档与代码保持一致。

## 规则 2：每次修改都要写入修改历史

每次进行了什么修改，都要在 **`修改历史.md`**（位于本工作文件夹 `D:\AAA_develop\github\pixel_animator-1.8-UI-\修改历史.md`）文末追加一条记录，写明：

- **被更改的文件**（路径）
- **更改日期**
- **修改大致内容**

追加位置：始终写在文件最末尾，按时间顺序向下延伸，便于追溯。

## 规则 3：新功能必须能被撤销

若添加了新的功能（无论影响像素、帧、图层还是画布尺寸/视图等），都要让**撤销（Ctrl+Z）/ 重做（Shift+Ctrl+Z）能够作用到该新功能上**。

- 实现要求：新功能在完成变更后，应调用全局快照 `PA.pushSnapshot()`（位于 `public/js/app.js`），使该操作进入撤销栈。
- 若新功能带有需要恢复的状态（如某种设置、尺寸、选中项），请在 `saveSnapshot()` / `restoreSnapshot()`（`app.js`）中一并保存与还原，保证撤销后能完整回到操作前状态。
- 矩阵：绘制、增删帧、复制/粘贴帧与图层、清空、像素翻转/旋转、画笔大小、图层增删/排序/合并/可见性/不透明度等均已接入撤销；新增功能请勿遗漏。
